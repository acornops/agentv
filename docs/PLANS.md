# Plans

Execution plans live in:

- `docs/exec-plans/active/`
- `docs/exec-plans/completed/`
- `docs/exec-plans/tech-debt-tracker.md`

Create a plan when work spans multiple files, multiple decisions, public contracts, host access boundaries, deployment behavior, or more than one repository.

## Plan Requirements

Each plan should include:

- goal
- scope
- affected files or modules
- contract impact
- validation commands
- rollout or rollback notes
- open questions
- completion summary
